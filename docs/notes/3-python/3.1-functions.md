## Create a Function

We've seen how to call existing functions. Now let's see how to create our own functions.

In Python, use the `def` keyword to create a function:

```python
def my_function():
    print("This is my function!")
```

- `def` indicates the start of a function definition.
- The name of the function comes next (similar naming runs apply as with variables).
- Use empty parentheses `()` if there are no arguments.
- Must have a colon (`:`) at the end of the line.
- Any code inside the function must be indented.

If you want your function to have parameters, specify the names of the parameter variables inside the parentheses:

```python
def my_function(param1, param2, param3):
    print("my_function called with parameters:", param1, param2, param3)
```

You can use the parameter variables just like regular variables you've been using.

When you run a Python script, code is executed from top to bottom. However, the code inside a function is only called when that function is called.

If the only thing in your Python file is a function, running that script won't do anything.

```python
def my_function():
    print("This is my function!")
```

You can call a function only _after_ it has been defined:

```python
def my_function():
    print("This is my function!")

my_function()
```

- Try calling the function before its defined and see what happens.
- Reminder that correct indentation is required in Python!
- Try stepping through a function with a debugger to observe the flow of execution.

## Return Values

It is often useful to have a function return a value back to whoever called the function. Some functions return a value, some don't.

The `print()` function does not return any value. In Python, it returns the special `None` value.

```python
val = print("Hello, world")
print(val)
```

So with `print()`, we choose to ignore this return value:

```python
print("Hello, world")
```

Now consider the `max()` function:

```python
val = max(1, 7, 5, 3)
print(val)
```

The `max()` function executes some set of instructions and returns a value. The caller in this example stores the return value into the `val` variable.

You can think of the return value of the function replacing the function call after it has been made.

For example:

```python
val = 2 * max(1, 3, 5) + 8
print(val)
```

When this code is executed, the call to `max(1, 3, 5)` is executed first. The return value is substituted back into the expression:

```python
val = 2 * 5 + 8
print(val)
```

You can make multiple function calls in a single expression, and even nest function calls:

```python
from math import sqrt
print(max(2, 4) + min(2, 8) + abs(-25.2) - sqrt(max(16, 64)))
```

When creating a function, how do you specify a return value? Use the `return` statement:

```python
def times_two(a):
    return a * 2
```

You can have multiple return statements in a function:

```python
def get_points(action):
    if action == "2pt":
        return 2
    if action == "3pt":
        return 3
    if action == "ft":
        return 1

    return 0
```

- `return` will return from the current function.
- You have to use `return` from within a function.
- `return` can be used within branches; it will do the same thing.

If a function does not have a return value, you don't have to use the `return` statement:

```python
def execute(timer):
    print("Executing action...")
```

However, sometimes you make want to exit a function early. You can still use the `return` statement, just don't specify any value to return:

```python
def execute(timer):
    if timer > 30:
        return

    print("Executing action...")
```

### Return vs Print

We've said that a program or function might "output" something to the console. This is different from saying a program or function will "return" a value.

The `print()` function tells the operating system to display characters in the console window. However, the function itself _returns_ nothing (`None` as we saw earlier).

The `max()` function does not display any characters in the console window as output. It does _return_ a value that can be used by the calling code.

Consider this function:

```python
def do_a_thing():
    print("Starting the thing...")
    print("Finished!")
```

- `do_a_thing()` prints two lines of text to standard output.
- It does not return any value.

Consider this modified version of the same function:

```python
def do_a_thing(status):
    if status == "bad":
        return False

    print("Starting the thing...")
    print("Finished!")
    return True
```

- `do_a_thing()` now returns either `True` or `False` indicating if the action was successful.
- It might still print lines of text to standard output.

Hopefully it is clear the difference between printing characters to standard output and a function returning a value.

## Function Comments

A function is an example of abstraction. A function provides a service to whoever uses it. You call a function, provide input, and the function provides output.

Knowing what a function does, what parameters it has, and what it returns is important.

Good documentation is vital to communicate this information.

In Python, you will typically see function documentation comments underneath the definition line and insight a pair of triple double quotes. Here's an example:

```python
def turn_on_light():
    """
    Turns on the current light bulb. If the bulb is already on, no change is made.
    """

    print("Turning on light...")
    # Additional code here...
```

For functions that are part of published packages, documentation comments can get fairly extensive.

Here is an example of a [function](https://github.com/daviddrysdale/python-phonenumbers/blob/facd74c8c2aaf5a6af56a92f037fba2e466eb96a/python/phonenumbers/phonenumberutil.py#L3262-L3293) from the [phonenumbers](https://github.com/daviddrysdale/python-phonenumbers) Python library:

```python
def is_number_match(num1, num2):
    """Takes two phone numbers and compares them for equality.

    For example, the numbers +1 345 657 1234 and 657 1234 are a SHORT_NSN_MATCH.
    The numbers +1 345 657 1234 and 345 657 are a NO_MATCH.

    Arguments
    num1 -- First number object or string to compare. Can contain formatting,
              and can have country calling code specified with + at the start.
    num2 -- Second number object or string to compare. Can contain formatting,
              and can have country calling code specified with + at the start.

    Returns:
     - EXACT_MATCH if the country_code, NSN, presence of a leading zero for
       Italian numbers and any extension present are the same.
     - NSN_MATCH if either or both has no region specified, and the NSNs and
       extensions are the same.
     - SHORT_NSN_MATCH if either or both has no region specified, or the
       region specified is the same, and one NSN could be a shorter version of
       the other number. This includes the case where one has an extension
       specified, and the other does not.
     - NO_MATCH otherwise.
     """
    if isinstance(num1, PhoneNumber) and isinstance(num2, PhoneNumber):
        return _is_number_match_OO(num1, num2)
    elif isinstance(num1, PhoneNumber):
        return _is_number_match_OS(num1, num2)
    elif isinstance(num2, PhoneNumber):
        return _is_number_match_OS(num2, num1)
    else:
        return _is_number_match_SS(num1, num2)
```

The details of exactly how the phone numbers are matched is not relevant in most cases. You can read the documentation and understand what the function does and how to use it.

Writing comments for a function can also help you evaluate a function you just wrote. Does the function try to do too much? What is its purpose? Do the required parameters make sense? What does the function return if there is invalid input?

## Variable Scope

Variables have a lifetime. They don't last forever.

The lifetime of a variable is its _scope_. Variable scope is a little strange in Python. We won't go into great detail here, but we'll mention some things that are good to know.

Here's a very simple example:

```python
name = "Hello"   # <-- start of scope for `name`

# blah blah blah...

print(name)      # <-- can use variable later on in same block
```

Obviously you can't use a variable _before_ it is created.

```python
print(name)      # <-- Nice try
name = "Hello"
```

A variable defined inside a function is scoped to just that function:

```python
def do_a_thing():
    x = 2
    if x == 2:
        print("You win!")

do_a_thing()
print(x)
```

- This doesn't work because `x` doesn't exist outside the `do_a_thing()` function.

Consider this example:

```python
def do_a_thing():
    x = 2
    if x == 2:
        print("You win!")

x = 3
do_a_thing()
print(x)
```

- The variable `x` inside the function is a completely separate variable from the `x` outside the function.
