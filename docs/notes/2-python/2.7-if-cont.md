## Compound Conditions

Sometimes you want multiple things to be true before taking a particular branch:

```python
num = 5
if num < 10 and num > 0:
    print("In range.")
else:
    print("Out of range.")
```

You can use logical operators to combine different Boolean expressions:

```python
num = 5
if num > 10 or num == 5:
    print("Valid")
else:
    print("Invalid")
```

Another example:

```python
isSystemArmed = True
isMotionDetected = True

if isSystemArmed and isMotionDetected:
    print("Motion was detected. Alarm will sound.")
elif isSystemArmed:
    print("System is armed, but no motion detected.")
elif isMotionDetected:
    print("Motion detected, but system not armed.")
else:
    print("System not armed; no motion detected.")
```

## Nested Ifs

You can have nested branches of code.

```python
isSystemArmed = True
isMotionDetected = True

if isSystemArmed:
    print("System is armed.")
    if isMotionDetected:
        print("Motion detected.")
        print("Alarm will sound.")
    else:
        print("No motion detected.")
```

Pay close attention to the indentation! Proper alignment must be used or Python will complain (or possibly give you a result you didn't expect).

Compare the previous code with the following code. What's the difference?

```python
isSystemArmed = True
isMotionDetected = True

if isSystemArmed:
    print("System is armed.")
    if isMotionDetected:
        print("Motion detected.")
        print("Alarm will sound.")
else:
    print("No motion detected.")
```

You can create as many levels of nesting as you want. But things start to get hard to read.

Try making sense of this:

```python
x = 105
if x > 5:
    if x > 10:
        if x > 50:
            if x > 100:
                print("Greater than 100.")
        else:
            print("Not greater than 50.")
    elif x == 7:
        print("x is 7.")
```

## Early Exits

Consider this example from the textbook:

```python
from pcinput import getInteger

num = getInteger("Please enter a positive integer: ")

if num < 0:
    print("You should have entered a positive integer!")
else:
    print("Now I am processing your integer", num)
    print("Lots and lots of processing")
    print("Hundreds of lines of code here")
```

This program works fine. But sometimes it is cleaner to check for invalid conditions and exit or return early.

Consider this variation:

```python
from pcinput import getInteger

num = getInteger("Please enter a positive integer: ")
if num < 0:
    print("You should have entered a positive integer!")
    exit()

print("Now I am processing your integer", num)
print("Lots and lots of processing")
print("Hundreds of lines of code here")
```

This code does the same thing. But now the primary branch of code does not have to be indented inside an else block.

As programs get more complex, thinking about _readability_ of your code is important. Code that is hard to read can be difficult to maintain. It might lead to bugs.

We'll see a similar concept when we write our own functions (instead of `exit()` you use `return`).

### Improvements

**Pass 1**

- Variable names (they should make sense)
- Consistent indentation
- Consistent spacing
- Use blank lines to separate related blocks of code
- Unnecessary `()`

Many of these issues are fixed automatically with a code formatter like we use in our exercise repositories.

**Pass 2**

- Add comments to every block of code (related group of statements)
- Comments should help describe the purpose/intent of the code, not strictly repeating syntax

**Pass 3**

If you see four levels of nested if statements, you should at least raise an eyebrow and see if the code could be reorganized or simplified. Sometimes that level of nesting is acceptable. But often things can be improved.

The if/elif blocks could be re-organized so the outer conditions are age and the inner conditions are checking income levels. This way, only one level of nested if statements is required.

You might consider moving hard-coded numbers (such as age and income thresholds) into constants. These hard-coded numbers are considered "magic numbers". Just seeing the value itself doesn't provide any meaning for why that number exists.

```python
if income > 100000:
    income_tier = "very_high"
```

Compared to:

```python
VERY_HIGH_INCOME_THRESHOLD = 100000

if income > VERY_HIGH_INCOME_THRESHOLD:
    income_tier = "very_high"
```

This also let's you use that value in other places, making it easier to change the value later if needed.

At this point we are good enough. However, it's worth thinking through additional possible improvements.

This is going to be an advanced code evaluation, beyond basic code style. Writing good code is part science, part art. Being able to evaluate code on multiple levels takes time and practice.

Step back and consider the point of this program. The program classifies a user into a certain tier, then determines program eligibility. There are age tiers and income tiers.

We don't have the full requirements for this program. Perhaps the company this program was written for had strict requirements about their tiers. We'll pretend we have some flexibility for this exercise.

Question: do we need to have variations of income tiers within different age ranges?

For example, would it be bad to classify a senior as having "high" vs "very high" income? Currently, there is no "very high" classification for seniors.

If we could standardize income tiers, the code could be simplified further.

```python
# Get user input
age = int(input("Enter your age: "))
income = float(input("Enter your annual income: "))

# Determine age tier
if age > 65:
    age_tier = "senior"
elif age >= 18:
    age_tier = "adult"
elif age >= 13:
    age_tier = "teen"
else:
    age_tier = "minor"

# Determine income tier
if income > 100000:
    income_tier = "very_high"
elif income > 50000:
    income_tier = "high"
elif income > 0:
    income_tier = "low"
else:
    income_tier = "no_income"

# Output tier
print("Tier:", f"{age_tier}_{income_tier}")

# Output program eligibility
if age_tier == "senior":
    if "high" in income_tier:
        print("  -> Eligible for premium senior benefits")
    elif "low" in income_tier:
        print("  -> Eligible for assistance program")
else:
    print("  -> Standard processing")
```

There are many other directions we could take this code, depending on requirements, but this will do for now.
